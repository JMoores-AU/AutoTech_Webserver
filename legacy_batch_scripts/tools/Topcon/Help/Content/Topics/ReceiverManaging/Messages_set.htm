Message Set

The Message set is a customized collection of messages that can be enabled with a single command without specifying each message explicitly. A message set might comprise up to 63 messages from various groups. Using TRU you can add or remove messages or modify their scheduling parameters.

To edit a message set, right click on any place of the Logs tab, select the Default Message Set command in the context menu and select the desired message set from the list. The def <n> dialog is opened, where you can view and edit a list of messages being logged into the file.

There are five message sets shown in TRU:

  • Original default message set (/msg/def). In its original form, it comprises binary TPS messages designed to meet the requirements of various surveying software programs. It is the primary message set used by the receiver when you start data logging.
  • Default message set #1 (/msg/def1). In its original form, it comprises 10 binary BINEX records.
  • Default message set #2 (/msg/def2). In its original form, it comprises 5 binary RTCM 3.x messages.
  • Default message set #3 (/msg/def3). In its original form, it comprises 2 text messages: NMEA GGA and a TPS ASCII message CL.
  • Default message set #4 (/msg/def4). In its original form, it comprises no messages altogether.
