Receiver Message

In this dialog, you can specify the schedule of the message. The Schedule Specification fields are used to define the output scheme for the message:

  • Period - specifies the message output interval. If it is set to zero, the corresponding message will be output at the highest possible rate.
  • Phase - specifies a time shift in the output of a message with respect to the moment when the receiver time is a multiplier of the specified period.
  • Count- serves two purposes. Firstly, if set to a positive number, specifies how many times the message outputs. Zero means an unlimited number of times the message outputs. Secondly, if set to a negative number, instructs the receiver to insert the message into wrap-message (with >> ID) prior to output.
  • Flags - provides further adjustment of the way the message is output. They are briefly explained below.
Flag Description
Out If the flag is set, the first of the messages invoked by the corresponding command will be output at the epoch closest to the command execution time (regardless of the specified "period" option).
Change If the flag is set, the corresponding message will be output only if the message data have changed.
Out On Add If the flag is set, then the (first) message will be output immediately after executing the corresponding em or out command.
Notena If the flag is set, the receiver cannot output the file header every time you change the "period" value when recording data into the log file.
Fix Per If these flags are set, the user cannot change the option values (Period, Phase, Count, and Flags).
