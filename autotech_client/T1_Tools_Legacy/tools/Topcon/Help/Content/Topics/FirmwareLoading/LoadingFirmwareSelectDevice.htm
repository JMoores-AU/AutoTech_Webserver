Select Target

To load the firmware to a device, do the following in this dialog:

  1. In the Target control, select the device to which you want to load the firmware. The following device types are available:
    • Receiver,
    • Peripheral Device (Modem, Bluetooth, etc.),
    • Power Board,
    • MC Controller Board,
    • MC IMU
  2. In the Firmware Image Format control, you can specify which firmware format to use:
    • There are following firmware formats for the Receiver:
      • *. ldp or *.tfi file. Using this selection you can load a GNSS firmware file to a receiver, which has a Flash Card.
      • *. ldr and *.ldp files. Using this selection you can load two GNSS firmware files to a receiver via physical Serial Port. In this case the receiver Flash card is not required. Also you can select the Capture Method:
        • Soft Break method - the receiver will be switched into firmware updating mode through the software.
        • Power ON method - the receiver will be switched into firmware updating mode after power reset of the receiver.

        When you are updating firmware of a TPS receiver integrated into a controller (GRS-1 and Tesla) using the Power ON capture method, you do not need to reset the receiver. The Application does it automatically.

    • There are following firmware formats for the Peripheral Device (Modem, Bluetooth, etc):
      • *. ldp or *.tfi file. Using this selection you can load a modem or Bluetooth firmware file to a receiver, which has a Flash Card.
      • *. ldr and *.ldp files. Using this selection you can load two a modem or Bluetooth files to a receiver via physical Serial Port. In this case the receiver Flash card is not required. Also you can select the Capture Method
      • Native firmware file. Using this selection you can load native modem firmware file to the modem. In this case, for the internal modem you can specify the serial port through which it is connected with the GNSS receiver (e.g Via Receiver's "Serial N"), and for external modem you can specify the direct connection to PC or Controller.
    • There are following firmware formats for the Power Board :
      • *.tfi file. Using this selection you can load the power board firmware file to a  receiver, which has a Flash Card.
      • *. ldr files. Using this selection you can load the power board firmware file to a receiver via physical Serial Port. In this case the receiver Flash card is not required. Also you can select the Capture Method
    • There is one firmware format for the MC Controller Board:
      • *. ldr and *.ldp files. You can load two MC Controller board firmware files to a receiver via physical Serial Port. In this case the receiver Flash card is not required and the Capture Method is always Power On.
    • There is one firmware format for the MC IMU:
      • *. ldp file. Using this selection you can load a MC IMU firmware file to a receiver, which has a Flash Card.
  3. Click Next to review the information about the connected device.
