Client

The Client tab allows you to configure connection between the rover receiver and an external NtripCaster:

• Enable check box— when checked, the NtripClient functionality for the receiver is enabled. If the functionality is activated, NtripServer functionality will be automatically disabled for Server or Server 1. The receiver can not perform NtripClient and NtripServer functions at the same time (for HiPer II, HiPer V, HiPer SR, GR-5 and Net-G3A receivers).
• Caster — specifies the DNS host name or IP address of an NtripCaster to which the receiver, acting as an NtripClient, will connect. Click to open the Caster dialog.
• Mountpoint Name — specifies an NtripCaster mountpoint where the receiver, acting as an NtripClient, will get the corrections data. We recommend to manually enter the desired name. Click to open the Source Table dialog.
• Protocol — defines the way of receiving correction data. The following selections are available from the drop- down list: Ntrip v1.0 , Ntrip v2.0 with HTTP or Automatic Selection.
• Input Mode — specifies the type of data to input on the Ntrip A receiver port. The following selections are available from the drop- down list: None, Command, Echo, Auto, RTCM 2.x, RTCM 3.x, CMR/CMR+, TPS, OmniSTAR, Dat Transfer or PPP.
• Current State — shows the current state of the parameter for Ntrip A receiver port.
• Last Error — reports the errors occurred while using Ntrip.

Click the User button to open User Credentials dialog, where you can specify the user name and password required by Ntrip.

Click the Proxy button to open Proxy Server dialog, where you can specify the IP address and TCP port number of a proxy server located between the NtripClient and NtripCaster.

Click the Advanced button to open Advanced Settings dialog, where you can specify extra parameters for the rover receiver.
