Source Table

The Source Table dialog shows the mountpoints list, which the rover receiver got from the NtripCaster. The list contains the following parameters:

• Mountpoint — specifies the NtripCaster mountpoint name.
• Identifier— specifies a name of city next to source location.
• Format — specifies a correction data format.
• Format details — specifies the format of messages: RTCM or raw data.
• Carrier — specifies carrier phase information:
  • No— the correction data does not contain the carrier (DGNSS solution only),
  • L1— the correction data contains L1 only (L1 RTK solution),
  • L1+L2— the correction data contains L1/L2 (L1/L2 RTK solution).
• Nav-system — specifies navigation system which is used for RTK/DGNSS solution.
• Network — specifies a network name.
• Country — contains three charters country code in ISO 3166.
• Latitude/Longitude — specifies the position of the GNSS antenna phase center.
• Requires NMEA — shows if it is necessary for the rover receiver to send the NMEA messages with approximate position to NtripCaster:
  • No — the rover receiver must not send NMEA message with approximate position to the NtripCaster,
  • Yes— the rover receiver must send NMEA GGA message with approximate position to the NtripCaster.
• Solution — specifies the stream generator type:
  • Single Base — a single base generates the correction data,
  • Network — a network generates the correction data.
• Generator — specifies the stream generator mode:
  • Receiver model is shown, if a receiver generates the correction data,
  • Software name is shown, if a software generates the correction data,
  • Unknown — if the correction data generator is unknown.
• Compression — shows a type of the compression / encryption algorithm
• Authentication — shows authentication parameters:
  • None — an access protection does not apply,
  • Basic — basic way is used for access protection,
  • Digest — digest way is used for access protection.
• Fee — shows about service fee:
  • None — user does not fee for receiving the correction data,
  • Yes — user fees for receiving the correction data.
• Data rate, bps — specifies the bit rate of data stream, bits per second.
• Miscellaneous — specifies miscellaneous information.
