SBAS

The SBAS tab allows configuring WAAS/EGNOS/MSAS/GAGAN/QZSS-SAIF in the GNSS receiver

To make the receiver track a SBAS satellite, select the corresponding SBAS PRN.

  • Track (PRN) – lists PRN’s of all SBAS satellites. Check mark corresponding boxes to allow the receiver to track selected SBAS satellites. Use the pop-up menu options to enable or disable tracking of all satellites.
  • Use for Positioning (PRN) – lists PRN’s of SBAS satellites that you can use for positioning. Check mark corresponding boxes to allow the receiver to use selected SBAS satellites. Use the pop-up menu options to enable or disable usage of all satellites.
  • Interpret message #0 as – enables and disables the receiver to use any data from SBAS satellites if message type 0 is being broadcast.
    • 0: the receiver will not substitute message type 0 and therefore will not use any data from the SBAS satellite.
    • [1...99]: the receiver will substitute the selected message type for message type 0 and therefore will interpret message type 0 as a message with the selected type.
  • Use Iono Corrections – enables and disables the receiver to apply ionospheric delay corrections broadcast by a SBAS satellite to the GPS satellites:
    • None - the receiver will not apply ionospheric corrections;
    • Apply If Available - the receiver will apply the available ionospheric corrections to the corresponding GPS satellites. If iono corrections are unavailable for some satellites, the receiver will apply the standard Klobucher iono model for these satellites.
    • Use Sat Only If Available - the receiver will apply the available ionospheric corrections to the corresponding GPS satellites and only these satellites will be used when computing the code differential position. If the number of GPS satellites with ionospheric corrections is not enough for computing the code differential position, the receiver will produce a standalone position.

      • For none Vanguard, G3 and TG-3 based receivers, the SBAS page provides the setting True SBAS universal satellite number (USI). With this setting enabled, the receiver outputs true USIs in SI messages rather than associated with a GPS USI.

The ability to track and use SBAS depends on the receiver model, OAF, and firmware version you are running.

Right-click (PC) or tap (Mobile) anywhere in the dialog to bring up the following controls:

• Track all satellites – checks all checkboxes for all SBAS satellites in the Track (PRN) panel. After that the GNSS receiver will track all SBAS satellites.
• Don't track any satellites – unchecks all checkboxes for all SBAS satellites in the Track (PRN) panel. After that the GNSS receiver will not track the SBAS satellites and these satellites will not be used in the positioning.
• Use all satellites – checks all checkboxes for all SBAS satellites in the Use for Positioning (PRN) panel. After that the GNSS receiver will use in the positioning all tracked SBAS satellites.
• Don't use any satellites – unchecks all checkboxes for all SBAS satellites in the Use for Positioning (PRN) panel. After that the GNSS receiver will not use the SBAS satellites in the positioning.
