Data Link

The Data Link tab displays the live statistics of the data communication links used for receiving DGNSS/RTK measurements from various sources, including physical reference stations, network software such as TopNET, Ntrip casters, etc.

Use this page to determine data loss and link viability. Based on this statistics you can see how well your data communication link is performing.

To analyze the data use the following parameters:

  • Port – a name of a receiver port, which obtains DGNSS/RTK corrections.
  • Decoder – a name of format of incoming DGNSS/RTK corrections to the corresponding receiver port.

    • It can have the following values:

    • RTCM 2.x / RTCM3.x /CMR and other corrections format – the format correction name, which the receiver's port obtains.
    • Detecting - the corresponding port is set to the Auto mode, the receiver is trying to determinate a format of incoming corrections,
    • <correction format name> – the corresponding port is set in the Auto mode, the correction format is decoded, but the receiver does not use the correction. Such case can be occurred if the receiver obtains the RTCM 3.x corrections for two ports simultaneously, but OAF allows to use only one port for this correction format:

  • Ref Station – ID of a base station transmitting DGNSS/RTK corrections.
  • Link Quality, % – data link quality of the selected channel in percentage.
  • Data Link Delay, sec – time elapsed since receiving the last message, in seconds. The maximum value – 999 seconds.
  • Received Messages – number of received messages (between 0001 and 9999).
  • Corrupted Messages – number of corrupted messages (between 0001 and 9999).
