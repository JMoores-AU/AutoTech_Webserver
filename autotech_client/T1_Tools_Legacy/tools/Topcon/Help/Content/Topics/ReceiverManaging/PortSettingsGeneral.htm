Port Settings: General

For Serial, USB, TCP, Bluetooth, and Ntrip ports, you can change Input Mode, Output Mode, Period and Elevation Mask. You can also view and edit set of messages, which are output to the port.

• Input Mode- specifies the type of data to input on the selected port.
  • None - the port will reject any incoming data
  • Command- the port is in command mode and will recognize user-sent commands
  • Echo - the port is in echo mode and will redirect all incoming data to an output stream, either another port or the current log file
  • Auto - the port is in the mode of automatic detection of correction data format.

    • You can set the Auto mode only for one port of the receiver.

  • RTCM 2.x - the receiver recognizes and decodes RTCM version 2.1, 2.2, and 2.3 messages accepted on the corresponding port
  • RTCM 3.x - the receiver recognizes and decodes RTCM version 3.x messages accepted on the corresponding port
  • CMR - the port recognizes and decodes accepted CMR messages
  • TPS - the port recognizes and decodes accepted TPS messages
  • OmniSTAR - the port recognizes and decodes OmniSTAR VBS corrections
  • Data Transfer - the port is in Data Transfer Protocol mode, which is used for data transfer to/from the receiver
  • PPP - the port is in PPP mode

To be able to connect to the receiver through a port, input mode of the port should be Command.

You may not modify input mode of the port, which you are currently using to connect to the receiver.

A predefined message set can include messages with variable period as well as messages with fixed period. Messages with variable period will be generated by the receiver at the period specified in the Period, s edit box. Value entered to the Period, s edit box has no effect on messages with fixed period.
• Period, s- specifies the interval at which the receiver will generate messages included into the currently selected predefined message set, in seconds. This field will be enabled, if the port is set in the Output Mode.
• Elev. Mask, deg - specifies the minimum angle at which the receiver will output measurement data from satellites to a specified port. The receiver will not output measurement data to the specified port from the satellites with elevations below this mask. Measured in degrees. This field will be enabled, if the port is set in the Output Mode.
• Messages- opens the Messages dialog to configure output messages on the corresponding port manually.
• On the Ntrip tab for HiPer HR and Net G5 receivers the Input mode is not available for the followings ports: Ntrip B, Ntrip C, Ntrip 1, Ntrip 2, Ntrip 3, Ntrip 4, Ntrip 5, Ntrip 6, Ntrip 7, Ntrip 8, Ntrip 9. But for Ntrip A port both Input mode and Output mode are available.
