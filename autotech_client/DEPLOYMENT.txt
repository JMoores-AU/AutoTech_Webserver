============================================
  AUTOTECH CLIENT - DEPLOYMENT NOTES
============================================

DEPLOYMENT FLOW:
----------------
1. Development Environment (C:\AutoTech_WebApps\T1_Tools_Web-1\)
       |
       | BUILD_WEBSERVER.bat (option 11 or 13)
       v
2. USB Drive (E:\AutoTech\)
       |
       | User runs Install_AutoTech_Client.bat
       v
3. Remote Client PC (C:\AutoTech\)


WHAT GETS DEPLOYED TO USB:
--------------------------
From BUILD_WEBSERVER.bat:
  - AutoTech.exe              -> E:\AutoTech.exe
  - autotech_client\          -> E:\AutoTech\autotech_client\
    - Install_AutoTech_Client.bat
    - VERSION
    - tools\*.exe
    - scripts\*.bat
  - tools\equipment_db.py     -> E:\AutoTech\tools\
  - tools\ptx_uptime_db.py    -> E:\AutoTech\tools\
  - database\ folder          -> E:\AutoTech\database\


WHAT GETS INSTALLED ON CLIENT:
------------------------------
From Install_AutoTech_Client.bat:
  - tools\*.exe               -> C:\AutoTech\
  - VERSION                   -> C:\AutoTech\
  - scripts\*.bat             -> C:\AutoTech\scripts\
  - mms_scripts\*.bat         -> C:\AutoTech\mms_scripts\
  - URI handlers registered in Windows Registry


IMPORTANT NOTES:
----------------
1. USB must be connected during build to deploy changes

2. Changes to client tools/scripts are NOT automatically
   pushed to remote clients. Users must:
   - Run Install_AutoTech_Client.bat from USB again
   - OR manually copy updated files

3. VERSION file is used to track updates:
   - Server checks autotech_client\VERSION
   - Client has C:\AutoTech_Client\VERSION
   - Mismatch shows "Update Available" in dashboard

4. To update client version:
   - Edit autotech_client\VERSION (e.g., change 1.1.1 to 1.1.2)
   - Run build to deploy to USB
   - Users re-run installer to get update


REMOTE PC SETUP (CRITICAL):
----------------------------
SCENARIO: Server at 10.110.19.105:8888, Remote PC at 10.110.19.100

PROBLEM: VNC opens on SERVER instead of REMOTE PC

SOLUTION: Install client on REMOTE PC
  1. Copy autotech_client folder to USB
  2. On REMOTE PC: Run Install_AutoTech_Client.bat as admin
  3. This installs to C:\AutoTech_Client
  4. Registers URI handlers (autotech-vnc://, etc.)
  5. Now VNC opens LOCALLY on remote PC

HOW IT WORKS:
  - Web page sends: autotech-vnc://10.110.21.87:5900
  - Browser asks: "Open with AutoTech VNC Protocol?"
  - Windows launches: C:\AutoTech_Client\scripts\launch_vnc.bat
  - Script starts: C:\AutoTech_Client\vncviewer_5.3.2.exe
  - VNC opens on LOCAL machine (not server)

CLIENT MUST BE INSTALLED ON EACH REMOTE PC THAT ACCESSES SERVER


WINDOWS SERVICE INSTALLATION (SERVER PC):
------------------------------------------
To run AutoTech 24/7 as a Windows service (survives reboots, no user login):

LOCATION ON USB:
  E:\AutoTech\Install_AutoTech_Service.bat
  E:\AutoTech\Uninstall_AutoTech_Service.bat

INSTALLATION STEPS:
  1. On SERVER PC: Right-click Install_AutoTech_Service.bat
  2. Select "Run as administrator"
  3. Installer will find E:\AutoTech.exe automatically
  4. Choose "Y" to start service immediately

BENEFITS:
  - Runs 24/7 even when no user logged in
  - Auto-starts on Windows boot
  - Auto-restarts if crash occurs
  - No security warnings (uses built-in Windows sc.exe)

MANAGEMENT:
  sc start AutoTech      - Start service
  sc stop AutoTech       - Stop service
  sc query AutoTech      - Check status
  Uninstall_AutoTech_Service.bat - Remove service

TROUBLESHOOTING:
  - If port 8888 already in use: netstat -ano | findstr :8888
  - Check Event Viewer: Windows Logs > Application
  - Service won't start: Check AutoTech.exe path in service config
